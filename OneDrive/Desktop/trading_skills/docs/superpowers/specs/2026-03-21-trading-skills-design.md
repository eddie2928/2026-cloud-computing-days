# OpenClaw 주식 트레이딩 Agent 스킬 설계

## 개요

코스피/코스닥 레버리지·인버스 ETF 양방향 스캘핑 매매 전략을 OpenClaw Agent 스킬로 구현한다. 매수 스킬 1개 + 매도 스킬 1개, 총 2개의 스킬로 구성하며 하이브리드 방식(결정론적 규칙 + LLM 보조 판단)을 사용한다.

## 요구사항

- **플랫폼:** OpenClaw (SKILL.md 형식)
- **스킬 수:** 매수 1개 (`buy-signal`) + 매도 1개 (`sell-signal`)
- **실행 방식:** 완전 자동 (사용자 승인 없이 주문 실행)
- **범위:** 전략 로직(시그널 판단, 주문 파라미터 결정)에만 집중. 키움증권 API 연동은 별도 구현.
- **브로커:** 키움증권 Open API

## 용어 정의

- **예수금:** D+2 결제 완료된 인출 가능 현금 (미체결 주문 금액 및 미결제 매수 금액 제외)
- **총 노출 한도:** 레버리지 + 인버스 포지션의 평가금액 합계 상한 = 예수금의 100%
- **스킬 호출 주기:** 5분봉 캔들 마감 시점마다 1회 호출 (장중 09:00~15:20)

### 핵심 원칙

- **결정론적 레이어가 1차 판단** — PDF의 기술적 지표 조건을 규칙으로 평가
- **LLM 보조 레이어가 2차 조정** — 참고 시그널로 매매 강도(수량 비율)를 조절하거나, 위험 상황에서 매매를 보류
- **LLM은 결정론적 레이어의 "매수/매도 실행" 판단을 뒤집지 못함** — 수량을 줄이거나 보류를 권고할 수 있음 (안전장치)

## 매수 스킬 (buy-signal) 상세

### 결정론적 레이어 — 3가지 매수 모드

#### 모드 1: 일반 매수 (스캘핑)

- **기준 차트:** 5분봉
- **조건:** 파라볼릭 SAR 매수 신호(af:0.02, maxAF:0.2) **AND** RSI 과매도(Period:10, LPercent:30)
- **종목 선택:** 5분봉 시그널이 발생한 시장의 레버리지 ETF를 선택. 코스피 지수 기반 시그널 → KODEX 레버리지(122630), 코스닥 지수 기반 시그널 → 코스닥150레버리지(233740). 양쪽 동시 발생 시 거래량이 큰 쪽 우선.
- **수량:** 예수금의 50% 이내에서 산출
- **거래 빈도:** 1일 평균 8~10회

#### 모드 2: 물타기 매수 (추가 매수)

- **트리거:** 보유 종목 평가손실 10% 이상
- **수량:** 예수금의 30%
- **최대 물타기 횟수:** 종목당 1일 1회까지만 허용
- **레버리지 매수 조건:**
  - 스토캐스틱 슬로우(K:12, D:5) **주봉** 매수 신호
  - 파라볼릭 SAR(af:0.02, maxAF:0.2) **일봉** 매수 신호
  - 두 조건 동시 충족 시 실행

#### 모드 3: 인버스 헤지 매수

- **트리거:** 레버리지 포지션 평가손실 5% 이상 (물타기 트리거인 10%보다 먼저 헤지 진입)
- **대상:** KODEX 200선물인버스2X(252670) 또는 코스닥150선물인버스(251340)
- **조건:**
  - 스토캐스틱 슬로우(K:12, D:5) **주봉** 매도 신호
  - 파라볼릭 SAR **일봉** 매도 신호 **AND** 스토캐스틱 슬로우 **일봉** 83 이상
  - 두 조건 동시 충족 시 실행

### LLM 보조 레이어

결정론적 레이어가 매수 조건을 충족한 후, LLM이 다음을 평가하여 **매매 강도(수량 비율)를 50%~100% 범위에서 조절**:

| 참고 시그널 | 평가 내용 |
|---|---|
| 전일 미국 주식 (나스닥, S&P500) | 미국 시장 급락 시 수량 축소 |
| 외국인/기관 거래 대금 및 거래량 | 대규모 매도 시 수량 축소 |
| 데일리 시황 브리핑 | 악재 뉴스 시 수량 축소 |
| 기타 뉴스 자료 | 시장 충격 이벤트 감지 시 보류 |

**제약 조건:** LLM은 수량을 50% 미만으로 줄이거나 매수 자체를 거부할 수 없음 (최소 50% 실행 보장)

## 매도 스킬 (sell-signal) 상세

### 결정론적 레이어 — 2가지 매도 모드

#### 모드 1: 익절 매도 (기본)

- **조건:** 보유 종목의 현재가가 평균 매수가 대비 **+0.6% 이상** 상승
- **수량:** 전량 매도
- **실행:** 조건 충족 즉시 자동 매도 지시
- **대상:** 레버리지/인버스 구분 없이 보유 중인 모든 종목에 개별 적용

#### 모드 2: 목표수익률 도달 매도

- **조건:** 당일 총 실현 수익이 거래대금의 **0.4% 이상** 달성
- **동작:** 미체결 매수 주문 취소, 신규 매수 중단 신호 발생
- **보유 중인 포지션은 모드 1(0.6% 익절) 조건에 따라 개별 청산**

### LLM 보조 레이어

| 상황 | LLM 판단 |
|---|---|
| 시장 급락 감지 (지수 -2% 이상 등) | 레버리지 포지션 즉시 청산 권고 (손절) |
| 장 마감 임박 (14:50 이후) | 미익절 포지션 정리 권고 |
| 급등 모멘텀 감지 | 익절 기준을 0.6% → 1.0%로 상향 제안 |
| 대규모 외국인/기관 매도 전환 | 보유 포지션 조기 청산 권고 |

**제약 조건:**

- 0.6% 익절 매도는 결정론적 레이어가 즉시 실행하며, LLM 보조 레이어는 이 판단을 지연시킬 수 없음
- LLM의 익절 기준 상향은 **0.6% 도달 전에만** 사전 설정 가능 (예: 장 시작 시 또는 이전 사이클에서). 0.6% 도달 후에는 무조건 즉시 매도.
- 익절 상향 범위: 0.6% ~ 1.5% (LLM이 사전에 설정한 값)
- LLM의 손절/조기 청산 권고는 별도 플래그로 매수 스킬에 전달 (매수 중단 연동)

### 결정론적 레이어 추가: 장 마감 강제 청산

- **15:10 이후:** 신규 매수 전면 중단 (매수 스킬 비활성화)
- **15:20:** 미익절 보유 포지션 전량 시장가 매도 (결정론적, LLM 조정 불가)
- 이 규칙은 모든 다른 매도 조건보다 우선함

### 매수 스킬과의 연동

스킬 간 플래그는 `trading_state.json` 파일을 통해 전달 (OpenClaw 워크스페이스 내 공유 파일):

```json
{
  "daily_target_reached": false,
  "market_risk": false,
  "stop_loss_triggered": false,
  "eod_liquidation_mode": false,
  "circuit_breaker": false,
  "llm_take_profit_override": 0.6,
  "last_updated": "2026-03-21T10:35:00+09:00"
}
```

```
매도 스킬 → trading_state.json → 매수 스킬 읽기
매도 스킬 → [당일 목표 달성 플래그] → 매수 스킬: 신규 매수 중단
매도 스킬 → [시장 위험 플래그]     → 매수 스킬: 수량 최소화(50%)
매도 스킬 → [손절 실행 알림]       → 매수 스킬: 물타기 모드 진입 판단
```

## OpenClaw 스킬 파일 구조

### 디렉토리 구조

```
trading_skills/
├── buy-signal/
│   ├── SKILL.md              # 매수 스킬 정의
│   └── references/
│       └── strategy.md       # PDF 전략 요약 (Agent 참조용)
├── sell-signal/
│   ├── SKILL.md              # 매도 스킬 정의
│   └── references/
│       └── strategy.md       # PDF 전략 요약 (Agent 참조용)
└── [원본 PDF]
```

### SKILL.md 프론트매터

```yaml
---
name: buy-signal  # 또는 sell-signal
description: "코스피/코스닥 레버리지·인버스 ETF 양방향 매매 - 매수(또는 매도) 시그널 분석 및 주문 파라미터 결정"
version: 0.1.0
metadata:
  openclaw:
    requires:
      env:
        - KIWOOM_APP_KEY
        - KIWOOM_APP_SECRET
      bins:
        - python3
    primaryEnv: KIWOOM_APP_KEY
    emoji: "📈"  # 매수: 📈, 매도: 📉
---
```

```json
  "flags": {
```

## 자금 관리 규칙 (양쪽 스킬 공통)

| 규칙 | 값 |
|---|---|
| 총 거래대금 한도 | 예수금의 50% |
| 총 노출 한도 | 예수금의 100% (레버리지+인버스 합산) |
| 물타기 매수 금액 | 예수금의 30% |
| 물타기 트리거 | 종목별 평가손실 10% 이상 |
| 물타기 최대 횟수 | 종목당 1일 1회 |
| 1회 익절 기준 | +0.6% (LLM 사전 조정 시 최대 1.5%) |
| 당일 목표 수익률 | 거래대금의 0.4% |
| **당일 최대 손실 한도** | **예수금의 -2% (서킷 브레이커)** |
| LLM 수량 조절 범위 | 50%~100% (매수) |
| 장 마감 강제 청산 | 15:20 전량 시장가 매도 |
| 신규 매수 차단 시각 | 15:10 |

### 서킷 브레이커 (당일 최대 손실 한도)

당일 실현 손실 + 미실현 평가 손실 합계가 예수금의 **-2%** 에 도달하면:
1. 모든 미체결 매수 주문 즉시 취소
2. 신규 매수 전면 중단
3. 보유 포지션 전량 시장가 매도
4. 당일 매매 종료 (`trading_state.json`에 `circuit_breaker: true` 기록)

이 규칙은 결정론적이며 LLM이 무시하거나 조정할 수 없음.

## 데이터 입력 인터페이스

스킬은 기술적 지표 원시값을 직접 계산하지 않는다. 외부에서 계산된 지표 데이터를 입력으로 받는다.

### 필수 입력 데이터 (키움 API 또는 별도 서비스에서 제공)

| 데이터 | 소스 | 형식 |
|---|---|---|
| 5분봉 OHLCV | 키움 API (opt10080) | 시가, 고가, 저가, 종가, 거래량 |
| 파라볼릭 SAR 값 | python `ta` 라이브러리로 계산 | SAR 값, 매수/매도 신호 boolean |
| RSI 값 | python `ta` 라이브러리로 계산 | RSI 수치 (0~100) |
| 스토캐스틱 슬로우 | python `ta` 라이브러리로 계산 | %K, %D 값, 매수/매도 신호 |
| 보유 포지션 정보 | 키움 API (opw00018) | 종목코드, 수량, 평균매수가, 평가손익률 |
| 예수금 (D+2) | 키움 API (opw00001) | 원화 금액 |
| 당일 실현 손익 | 키움 API | 원화 금액 |

### 참고 입력 데이터 (LLM 보조 레이어용)

| 데이터 | 소스 | 갱신 주기 |
|---|---|---|
| 전일 미국 시장 지수 | 뉴스 API 또는 웹 스크래핑 | 장 시작 전 1회 |
| 외국인/기관 매매 동향 | 키움 API (opt10059) | 5분마다 |
| 시황 뉴스 | 뉴스 API (네이버 증권 등) | 30분마다 |

## 오류 처리 및 장애 대응

| 장애 상황 | 대응 |
|---|---|
| 시세 데이터 30초 이상 지연/미수신 | 신규 매수 중단, 기존 익절 주문 유지 |
| 키움 API 연결 끊김 | 모든 신규 주문 중단, 재연결 시도 (최대 3회) |
| LLM API 타임아웃 (5초 초과) | LLM 보조 레이어 스킵, 결정론적 레이어만으로 실행 (수량 100%) |
| 주문 거부/실패 | 해당 주문 로그 기록 후 다음 사이클로 넘어감 |
| 부분 체결 | 미체결 잔량은 다음 사이클에서 재평가 (기존 주문 취소 후 새 조건으로 판단) |
| `trading_state.json` 읽기 실패 | 안전 모드: 신규 매수 중단, 익절 매도만 실행 |

모든 스킬 호출은 `trading_log.jsonl` 파일에 기록:

```json
{
  "timestamp": "2026-03-21T10:35:00+09:00",
  "skill": "buy-signal",
  "cycle": 15,
```

## LLM 호출 방식

LLM 보조 레이어는 **비동기(async)** 로 동작한다:
- 결정론적 레이어의 판단은 즉시 실행 준비
- LLM 호출은 병렬로 진행 (최대 대기 5초)
- LLM 응답이 5초 내 도착하면 수량 조절 반영 후 주문 실행
- LLM 응답이 5초 초과 시 결정론적 레이어 결과 그대로 실행 (수량 100%)
